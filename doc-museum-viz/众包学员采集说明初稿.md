# 博物馆展陈可视化采集任务说明

## 一、项目背景

博物馆展览中，除了文物、展品和文字说明，还经常会出现地图、时间线、图表、信息图、互动屏、数字装置等内容。这些内容能够帮助观众理解展览中的时间、地点、人物、事件、工艺流程和知识关系，是展览叙事的重要组成部分。

本次采集任务关注的就是这些“展陈中的可视化”。请各位学员参观一个博物馆展览，重点记录展览中用于解释信息、组织信息或辅助观众理解内容的可视化，并通过在线网站完成拍照和文字记录。

采集结果将用于后续整理博物馆展陈可视化案例，分析不同博物馆如何通过视觉方式呈现知识、组织展览和引导观众理解内容。

## 二、采集任务范围

每位学员需要选择一个博物馆中的一个展览开展采集。一次采集任务对应一个展览。（任务冲突的最小单位是展览，不是博物馆；同一座博物馆可以有多位学员采集，但不能重复采集同一个展览）

在采集展陈中的可视化时，需要重点关注的对象包括但不限于：

- 地图：历史地图、路线图、分布图、展区导览图等。
- 时间线：历史年表、人物生平线、事件发展线等。
- 图表：柱状图、饼图、折线图、比例图、统计图等。
- 流程图：制作流程、工艺流程、制度流程、考古过程等。
- 关系图：人物关系、机构关系、文化传播关系、知识结构图等。
- 信息图：图标、数字、文字、图片组合而成的信息说明。
- 数据墙：集中展示大量数字、年份、地点、名单或统计信息的墙面。
- 互动屏：触摸屏、查询屏、互动地图、数字沙盘等。
- 实体装置：通过模型、灯光、机械结构、地面图形等方式表达信息的装置。

一般不需要记录的对象包括：

- 单独一件文物或展品。
- 只有普通说明文字的展板。
- 没有信息解释作用的装饰图案。
- 只用于展示照片、但没有分类、时间、地点或关系组织的普通照片墙。

如果现场情况不确定，可以先记录下来，并在文字描述中说明你的判断依据。

## 三、采集的数据组织结构

本次采集任务以一个展览为单位。这个展览位于某一座博物馆中，因此需要先填写博物馆名称、地址和展览名称。进入展览后，再按照现场策展空间和内容，将展览分成若干个“展陈单元”。每个展厅单元中，再记录其中出现的可视化项。

“展厅单元”可以理解为展览内部的一个相对完整的空间或主题部分。它可能是一个独立展厅，也可能是展览中的一个章节、一个区域或一段参观动线。划分展厅单元的目的，是为了说明每个可视化项具体出现在展览的哪个位置、处在什么样的展陈环境中。

## 四、基础信息填写要求

在系统内各位学员需要填写的基础信息内容包括：收集人姓名、收集人单位、联系方式、调研时间、所在城市、博物馆名称、博物馆地址、展览名称、展览举办时间、展览在线链接和展览介绍等。

填写要求：

- 博物馆名称和展览名称尽量使用现场正式名称。
- 展览举办时间如为常设展，可填写“常设展”。
- 展览在线链接可填写官网页面、公众号文章、购票页或其他公开介绍页面。
- 展览介绍不用太长，说明展览主题、主要内容和展品类型即可。

## 五、展厅单元填写要求

展厅单元是展览中的一个相对完整的空间或主题部分。请根据现场展览结构进行划分。如按现场展厅名称划分，如“序幕厅”“今日北大展厅”。或按展览章节划分，如“第一部分 起源”“第二部分 发展”。

划分时请注意：

- 不要把整个大型展览全部写成一个单元，否则后续很难定位具体内容。
- 不要把每个展柜都单独拆成一个单元，除非该展柜本身就是一个完整主题区。
- 如果现场已经有明确章节标题，优先使用现场标题。

每个展厅单元需要填写以下内容：

| 填写内容 | 填写要求 |
|---|---|
| 单元名称 | 优先使用现场展厅名、章节名或导览图上的名称；如果现场没有明确名称，可以根据空间位置或主题自行概括。 |
| 单元描述 | 说明该单元在展览中的位置、空间环境和主要内容。建议写清楚：这个单元位于哪里，主要展示什么内容，观众从哪里进入、往哪里走，灯光、人流、空间大小、展柜分布如何，可视化内容主要出现在什么位置。 |
| 环境照片 | 用于说明可视化项所在的空间位置，不是为了拍单个展品。建议拍摄单元入口或整体空间、主要展线方向，以及可视化项与展柜、展品、墙面、观众动线之间的关系；每个展厅单元建议至少拍 1-3 张环境照片。 |

## 六、可视化项填写要求

可视化项是本次采集的核心。每发现一个符合条件的可视化内容，请单独建立一条记录，并重点完成文字描述。文字描述可以采用口述转录的方式完成，不要求写成正式文章，但最好按照下面四个角度说清楚。

| 描述角度             | 需要说明什么（不限于）                                                                                                                                                                                                                                                         |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 可视化本身           | - 可视化类型 —— 地图 / 时间线 / 柱状图 / 关系图 / 信息图 / 数据墙 / 互动屏 / 实体装置 …<br>- 内容主题 —— 它在讲什么内容、什么主题。<br>- 数据编码 —— 编码了哪些数据？用什么视觉通道(位置 / 长度 / 颜色 / 大小)来表达？                                                         |
| 可视化在展览中的功能 | - 带来什么新信息 —— 相比旁边的文字说明 / 实物展品，它多给了什么？<br>- 它在展览中起什么作用 —— 例如概览、导览、解释背景、展示关系、呈现变化、帮助比较等。                                                                                                                      |
| 和人的交互形式       | - 媒介与物理形态 —— 静态 / 动态 / 可交互；尺度大小；屏幕、投影、印刷还是实体装置。<br>- 交互方式与观众行为 —— 观众做什么(看 / 触屏 / 走进去 / 动手)；会停下还是路过。<br>- 空间关系与展线位置 —— 和周围真实展品的关系；在参观动线里的位置(入口 / 中段 / 高潮 / 出口)。         |
| 你对它的评价         | - 感官 / 审美反馈 —— 第一眼观感、好不好看、有没有冲击力。<br>- 是否易懂 —— 看不看得懂、要不要先验知识、有没有成功传达。<br>- 保存状态 —— 尤其针对物理可视化：有没有坏、屏幕黑没、褪色、失灵、缺损、内容过时。 <br>- 受众可达性 —— 对不同观众(小孩 / 专家 / 普通观众)友不友好。 |


口述或填写时，可以直接按顺序说：“从可视化本身来看……；从它在展览中的功能来看……；从和人的交互形式来看……；从我的评价来看……”。不一定四个角度都要写得一样长，但请尽量多记录现场能观察到的信息。非常欢迎您补充说出除上述角度外，任何与该可视化有关的感受、想法、建议等。

## 七、现场采集流程

建议按以下顺序完成采集：

1. 先在群内填写任务登记表，写明计划采集的博物馆和展览名称，确认该展览没有与其他同学重复。
2. 到馆后再次确认博物馆名称、地址和展览名称。
3. 进入该展览，先观察展览整体结构，判断可以分成几个展厅单元。
4. 进入每个展厅单元后，先拍环境照片，再填写单元名称和单元描述。
5. 在该单元内寻找可视化项。
6. 对每个可视化项依次拍摄位置照片、完整照片和细节照片。
7. 按“可视化本身、可视化在展览中的功能、和人的交互形式、你对它的评价”四个角度完成可视化项的文字描述。
8. 离开展厅单元前检查是否漏拍、漏填。
9. 完成全部展厅单元后，确认照片和文字记录已经整理完整。


采集信息时不要求写得学术，但必须清楚、具体、真实。应保证他人能够通过所提供的照片和描述，判断该可视化项的内容、位置、形式和展览作用。
